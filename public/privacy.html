2. Google Data - Integration with Google Sheets and Google Drive

2.1. Data Accessed

The OnlyFlow application requests access to the following Google data through OAuth 2.0:

  • Google Sheets API (scope: https://www.googleapis.com/auth/spreadsheets):
    • Access to create, edit and manage Google Sheets spreadsheets
    • Read and write data in spreadsheet cells
    • Creation of new spreadsheets and sheets within spreadsheets
  • Google Drive API (scope: https://www.googleapis.com/auth/drive.readonly):
    • Read-only access to list the user's existing spreadsheets
    • View basic spreadsheet metadata (ID, name, URL)
    • Does not access other Drive files, only Google Sheets spreadsheets

Important: The application does not access, read or modify other Google Drive data beyond Google Sheets spreadsheets. We do not access emails, documents, photos or any other type of file.

2.2. OAuth Authentication Flow

  1. Access request: When the user adds a "Spreadsheet" node in a workflow in MindOnlyFlow and clicks "Authenticate with Google", the system generates an OAuth 2.0 authorization URL.
  2. User consent: The user is redirected to Google's consent page, where they view the requested scopes and authorize access.
  3. Token acquisition: After authorization, Google returns an authorization code that is exchanged for access tokens (access_token and refresh_token).
  4. Secure storage: Tokens are stored in encrypted form in the database, associated exclusively with the user ID.
  5. Automatic renewal: The system uses the refresh_token to automatically renew the access_token when needed, without requiring new user authorization.

2.3. Use of Data

Google data is used exclusively for:

  • Creating new spreadsheets in the user's Google Sheets;
  • Inserting contact data, conversations and workflow information into spreadsheets created or selected by the user (including leads from WhatsApp or Typebot, functioning as nodes within MindOnlyFlow);
  • Listing existing spreadsheets for selection in the interface;
  • Reading metadata (name, ID, URL) for display on the platform;
  • Exclusive use within the OnlyFlow application itself, with no other purposes.

2.4. Data Sharing

We do not share Google data with third parties. Specifically:

  • Tokens and data accessed via Google APIs are not shared, sold or transferred;
  • Communication occurs directly between the OnlyFlow server and official Google APIs, with no intermediaries;
  • We do not sell, rent or share your Google data with advertising companies, data brokers or any other third party.

2.5. Data Storage and Protection

Storage:

  • Only OAuth tokens (access_token and refresh_token) are stored in the database, associated with the user ID;
  • Tokens are stored on a secure server with encryption in transit (HTTPS) and at rest;
  • OnlyFlow does not store the content of Google spreadsheets; data remains exclusively in the user's Google account.

Security Measures:

  • Tokens stored in encrypted form;
  • Access restricted to the user only;
  • Communication with Google APIs via HTTPS;
  • Tokens never exposed in logs or public interfaces;
  • Technical and organizational measures to protect against unauthorized access, alteration or destruction.

2.6. Data Retention and Deletion

Token retention: OAuth tokens are kept while the user uses the Google Sheets integration feature. The user can revoke access at any time through Google's security settings.

Token deletion:

  • Tokens are automatically removed when the user revokes access on Google;
  • The user may request deletion by contacting us at contato@onlyflow.com.br;
  • When the user deletes their OnlyFlow account, all associated tokens are automatically removed.

Data in spreadsheets: Since data is inserted directly into the user's Google spreadsheets, it remains under the user's full control. OnlyFlow does not keep copies of this data and cannot access it after token revocation.

Deletion process: To request deletion of stored tokens, the user may:

  • Revoke access directly on Google (recommended) at myaccount.google.com/permissions;
  • Contact our support by email at contato@onlyflow.com.br;
  • Delete their OnlyFlow account.

3. Other Data Collected

3.1. WhatsApp Data

We collect messages, contacts, groups and conversation history for CRM and automation functionality. This data is used exclusively to provide communication and relationship management features.

3.2. Meta/Instagram Data

When you connect your Instagram account via Meta OAuth, we collect profile information, direct messages, comments and interactions to provide keyword-based automations, automatic responses and DM sending.

You can revoke access at any time at facebook.com/settings.

3.3. OpenAI Data

For natural language processing and AI agents, we send conversations and context to OpenAI for processing. We do not store personal data on OpenAI. Processed data includes conversation messages and interaction history to provide contextual automatic responses, conversation memory, audio transcription and lead qualification.

For more information, see the OpenAI Privacy Policy.

3.4. Typebot Data

When you use the Typebot integration, we share conversation and lead information for processing in Typebot, functioning as nodes within MindOnlyFlow.

3.5. Registration Data

We collect information such as name, email, phone and other information provided during registration and use of the platform.

3.6. Usage Data

We collect information about how you use our platform, including access logs, actions performed and configuration preferences.
